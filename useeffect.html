React - useEffect

← Retour au menu

Le hook useEffect permet de gérer les effets secondaires dans un composant React. Il est utilisé pour les appels API, timers, événements ou synchronisation avec des systèmes externes.

Exercice 1 — useEffect au montage

Exécuter du code lorsque le composant est monté.

Théorie

Un tableau de dépendances vide [] indique que l'effet doit être exécuté uniquement au montage.

Code Exemple

// useEffect exécuté une seule fois au montage
useEffect(() => {
  console.log("component mounted"); // affiche dans la console au montage
}, []); // tableau de dépendances vide

useEffect(()=>{
 console.log("component mounted")
},[])

Exercice 2 — Dépendance

Exécuter un effet lorsque une valeur change.

Théorie

Le tableau de dépendances permet de déclencher l'effet uniquement quand les valeurs changent.

Code Exemple

// useEffect surveillant la variable count
useEffect(() => {
  console.log(count); // affiche la valeur actuelle de count
}, [count]); // déclenche à chaque modification de count

useEffect(()=>{
 console.log(count)
},[count])

Exercice 3 — Cleanup

Nettoyer un effet avant destruction du composant.

Théorie

Une fonction retournée par useEffect sert à effectuer un nettoyage (cleanup) avant le démontage.

Code Exemple

// useEffect avec cleanup
useEffect(() => {
  return () => {
    console.log("cleanup"); // exécuté lors de la destruction du composant
  };
}, []);

useEffect(()=>{

 return ()=>{
  console.log("cleanup")
 }

},[])

Exercice 4 — Timer

Utiliser un intervalle.

Théorie

setInterval peut être utilisé dans useEffect pour créer des timers.

Code Exemple

// Compteur qui s'incrémente toutes les secondes
useEffect(() => {
  const interval = setInterval(() => {
    setCount(c => c + 1); // incrémente count toutes les secondes
  }, 1000);

  return () => clearInterval(interval); // nettoyage à la destruction
}, []);

useEffect(()=>{

const interval = setInterval(()=>{
 setCount(c => c + 1)
},1000)

return ()=>clearInterval(interval)

},[])

Exercice 5 — Appel API

Récupérer des données depuis une API.

Théorie

useEffect est utilisé pour exécuter des appels réseau après le rendu du composant.

Code Exemple

// Récupération de données depuis une API
useEffect(() => {
  fetch("https://jsonplaceholder.typicode.com/users")
    .then(res => res.json())
    .then(data => setUsers(data)); // met à jour le state avec les données
}, []);

useEffect(()=>{

fetch("https://jsonplaceholder.typicode.com/users")
.then(res => res.json())
.then(data => setUsers(data))

},[])

Exercice 6 — localStorage

Sauvegarder un state.

Théorie

useEffect peut synchroniser les données avec localStorage à chaque changement de state.

Code Exemple

// Sauvegarde du state count
useEffect(() => {
  localStorage.setItem("count", count);
}, [count]); // déclenche à chaque modification de count

useEffect(()=>{
 localStorage.setItem("count",count)
},[count])

Exercice 7 — Charger localStorage

Initialiser un state depuis le stockage local.

Théorie

useEffect peut charger des données au démarrage du composant.

Code Exemple

// Récupération du state depuis localStorage
useEffect(() => {
  const saved = localStorage.getItem("count");
  if (saved) {
    setCount(Number(saved)); // conversion en nombre
  }
}, []);

useEffect(()=>{

const saved = localStorage.getItem("count")

if(saved){
 setCount(Number(saved))
}

},[])

Exercice 8 — Event listener

Écouter un événement global du navigateur.

Théorie

useEffect permet de gérer l'ajout et le retrait d'événements pour éviter les fuites mémoire.

Code Exemple

// Écoute du scroll
useEffect(() => {
  function handleScroll() {
    console.log(window.scrollY); // position verticale du scroll
  }

  window.addEventListener("scroll", handleScroll);

  return () => window.removeEventListener("scroll", handleScroll); // cleanup
}, []);

useEffect(()=>{

function handleScroll(){
 console.log(window.scrollY)
}

window.addEventListener("scroll",handleScroll)

return ()=>{
 window.removeEventListener("scroll",handleScroll)
}

},[])

Exercice 9 — Effets multiples

Utiliser plusieurs useEffect dans le même composant.

Théorie

Chaque useEffect peut gérer un effet indépendant, ce qui permet un code clair et modulaire.

Code Exemple

// Premier effet
useEffect(() => {
  console.log("mount");
}, []);

// Deuxième effet
useEffect(() => {
  console.log("count changed");
}, [count]);

useEffect(()=>{
 console.log("mount")
},[])

useEffect(()=>{
 console.log("count changed")
},[count])

Exercice 10 — Optimisation

Limiter les effets inutiles pour améliorer les performances.

Théorie

Un bon tableau de dépendances permet de déclencher les effets uniquement lorsque c’est nécessaire.

Code Exemple

// Exécuter l'effet uniquement quand user change
useEffect(() => {
  console.log(user);
}, [user]);

useEffect(()=>{
 console.log(user)
},[user])
