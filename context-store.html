React - useContext (Store)

← Retour au menu

Le hook useContext peut servir de store global pour gérer des données comme un stock de produits ou un panier dans React.

Exercice 1 — Créer un contexte

Initialiser un contexte pour le store global.

Théorie

createContext permet de partager un state global accessible par tous les composants enfants.

Code Exemple

// Création d'un contexte vide
import { createContext } from "react";

// StoreContext sera partagé entre composants
const StoreContext = createContext(null);

import { createContext } from "react"

const StoreContext = createContext(null)

Exercice 2 — Provider

Partager le store avec tous les composants enfants.

Théorie

Le Provider rend les données et fonctions accessibles dans l’arborescence des composants.

Code Exemple

// Composant Provider pour le store
function StoreProvider({ children }) {
  // State global pour les produits
  const [products, setProducts] = useState([]);

  return (
    
      {children} {/* rend les enfants avec accès au store */}
    
  );
}

function StoreProvider({children}){
 const [products,setProducts] = useState([])

 return (
   <StoreContext.Provider value={{products,setProducts}}>
     {children}
   </StoreContext.Provider>
 )
}

Exercice 3 — Ajouter un produit

Ajouter un produit dans le store global.

Théorie

setProducts permet de mettre à jour le tableau de produits en créant un nouveau tableau.

Code Exemple

function addProduct(product) {
  // ajoute un produit tout en conservant les existants
  setProducts([...products, product]);
}

function addProduct(product){
 setProducts([...products, product])
}

Exercice 4 — Supprimer un produit

Retirer un produit du store par son identifiant.

Théorie

filter permet de créer un nouveau tableau sans l’élément ciblé.

Code Exemple

function removeProduct(id) {
  // supprime le produit dont l'id correspond
  setProducts(products.filter(p => p.id !== id));
}

function removeProduct(id){
 setProducts(products.filter(p => p.id !== id))
}

Exercice 5 — Mettre à jour un produit

Modifier les informations d’un produit existant.

Théorie

map permet de remplacer un élément précis dans le tableau sans modifier les autres.

Code Exemple

function updateProduct(updated) {
  setProducts(products.map(p => p.id === updated.id ? updated : p));
}

function updateProduct(updated){
 setProducts(products.map(p => p.id === updated.id ? updated : p))
}

Exercice 6 — Ajouter au panier

Gérer un panier séparé dans le store.

Théorie

Un state supplémentaire peut stocker le panier des produits.

Code Exemple

// State pour le panier
const [cart, setCart] = useState([]);

function addToCart(product) {
  setCart([...cart, product]); // ajoute le produit au panier
}

const [cart,setCart] = useState([])

function addToCart(product){
 setCart([...cart, product])
}

Exercice 7 — Calculer le total

Sommer les prix des produits dans le panier.

Théorie

reduce permet de cumuler les valeurs d’un tableau pour obtenir un total.

Code Exemple

function getTotal() {
  return cart.reduce((sum, p) => sum + p.price, 0);
}

function getTotal(){
 return cart.reduce((sum,p) => sum + p.price, 0)
}

Exercice 8 — Filtrer les produits

Afficher seulement certains produits selon une catégorie.

Théorie

filter permet de sélectionner des éléments spécifiques dans le tableau.

Code Exemple

function filterCategory(category) {
  return products.filter(p => p.category === category);
}

function filterCategory(category){
 return products.filter(p => p.category === category)
}

Exercice 9 — Persistance

Sauvegarder le store dans localStorage pour conserver les données.

Théorie

useEffect permet de synchroniser le state avec localStorage automatiquement.

Code Exemple

useEffect(() => {
  localStorage.setItem("products", JSON.stringify(products));
}, [products]); // mise à jour à chaque changement

useEffect(()=>{
 localStorage.setItem("products", JSON.stringify(products))
},[products])
