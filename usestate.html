React - useState

← Retour au menu

Le hook useState permet de gérer l'état d'un composant React. Il permet de stocker des données et de mettre à jour l'interface lorsqu'elles changent.

Exercice 1 — State simple

Créer une variable d'état.

Théorie

useState retourne deux éléments : la valeur et une fonction pour la modifier.

Code Exemple

// Déclaration d'un state count initialisé à 0
const [count, setCount] = useState(0);

// count contient la valeur actuelle
// setCount permet de modifier count

Explication du code

- count est la valeur du state.
- setCount est la fonction pour mettre à jour cette valeur. 

const [count, setCount] = useState(0)

Exercice 2 — Afficher le state

Afficher la valeur d'un state dans le JSX.

Théorie

La valeur du state peut être affichée dans les accolades JSX.

Code Exemple

// Affichage de la valeur count dans un paragraphe
<p>{count}</p>

Explication du code

- Les accolades permettent d'insérer la valeur du state dans le JSX.
- Chaque mise à jour de count déclenche un re-render automatique. 

<p>{count}</p>

Exercice 3 — Incrémenter un compteur

Modifier un state avec un bouton.

Théorie

La fonction setState permet de modifier la valeur.

Code Exemple

// Bouton qui ajoute 1 au compteur
<button onClick={() => setCount(count + 1)}>
  Ajouter
</button>

Explication du code

- L'événement onClick déclenche une fonction fléchée.
- La fonction appelle setCount pour incrémenter le state. 

<button onClick={() => setCount(count + 1)}>
Ajouter
</button>

Exercice 4 — Décrémenter

Réduire la valeur d'un state.

Théorie

Le state peut être modifié avec n'importe quelle opération JavaScript.

Code Exemple

// Bouton qui retire 1 au compteur
<button onClick={() => setCount(count - 1)}>
  Retirer
</button>

Explication du code

- On utilise la même logique que pour l'incrémentation.
- L'opération count - 1 met à jour le state. 

<button onClick={() => setCount(count - 1)}>
Retirer
</button>

Exercice 5 — Reset

Remettre le state à sa valeur initiale.

Théorie

La fonction setState peut réinitialiser la valeur.

Code Exemple

// Bouton qui réinitialise count
<button onClick={() => setCount(0)}>
  Reset
</button>

Explication du code

- Appelle setCount avec la valeur initiale.
- Permet de remettre le compteur à zéro facilement. 

<button onClick={() => setCount(0)}>
Reset
</button>

Exercice 6 — Input contrôlé

Synchroniser un input avec un state.

Théorie

Un input contrôlé utilise value et onChange pour refléter le state.

Code Exemple

// Déclaration du state
const [text, setText] = useState("");

// Input contrôlé
<input
  value={text}              // valeur liée au state
  onChange={(e) => setText(e.target.value)}  // met à jour le state
/>

Explication du code

- La valeur de l'input est toujours synchronisée avec text.
- OnChange capture les saisies et met à jour le state. 

const [text,setText] = useState("")

<input
value={text}
onChange={(e)=>setText(e.target.value)}
/>

Exercice 7 — Toggle

Basculer entre deux états.

Théorie

Un state booléen permet de gérer un affichage conditionnel.

Code Exemple

// Déclaration du state booléen
const [visible, setVisible] = useState(false);

// Bouton pour changer l'état
<button onClick={() => setVisible(!visible)}>
  Toggle
</button>

// Affichage conditionnel
{visible && <p>Bonjour</p>}

Explication du code

- !visible inverse la valeur actuelle.
- Le texte "Bonjour" n'est affiché que si visible est vrai. 

const [visible,setVisible] = useState(false)

<button onClick={() => setVisible(!visible)}>
Toggle
</button>

{visible && <p>Bonjour</p>}

Exercice 8 — State objet

Stocker un objet dans un state.

Théorie

Un state peut contenir un objet JavaScript.

Code Exemple

// Déclaration d'un state user
const [user, setUser] = useState({
  name: "Lucas",
  age: 25
});

// Affichage des propriétés
<p>{user.name} - {user.age}</p>

Explication du code

- user est un objet stocké dans le state.
- On peut modifier ses propriétés avec setUser. 

const [user,setUser] = useState({
 name:"Lucas",
 age:25
})

Exercice 9 — Tableau

Stocker une liste dans un state.

Théorie

Un tableau peut être utilisé pour afficher une liste dynamique.

Code Exemple

// Déclaration d'un state tableau
const [tasks, setTasks] = useState([]);

// Ajouter une tâche
setTasks([...tasks, "Nouvelle tâche"]);

Explication du code

- L'opérateur spread ... permet de conserver les anciennes valeurs.
- On ajoute une nouvelle tâche sans écraser l'ancien tableau. 

const [tasks,setTasks] = useState([])

setTasks([...tasks,"Nouvelle tâche"])

Exercice 10 — Mini Todo List

Créer une petite application avec useState.

Théorie

useState peut gérer une logique complète d'application simple.

Code Exemple

// States pour liste et nouvel input
const [tasks, setTasks] = useState([]);
const [task, setTask] = useState("");

// Fonction pour ajouter une tâche
function addTask() {
  setTasks([...tasks, task]);  // Ajoute la nouvelle tâche
  setTask("");                  // Reset du champ texte
}

Explication du code

- tasks contient toutes les tâches.
- task contient le texte saisi.
- addTask ajoute la tâche et réinitialise l'input. 

const [tasks,setTasks] = useState([])
const [task,setTask] = useState("")

function addTask(){
 setTasks([...tasks,task])
 setTask("")
}
