React - Props

← Retour au menu

Les props permettent de transmettre des données d’un composant parent vers un composant enfant dans React. Elles peuvent être des valeurs simples, des objets, des fonctions ou même des composants.

Exercice 1 — Première prop simple

Transmettre une chaîne de caractères à un composant enfant.

Théorie

Une prop est un paramètre passé à un composant. On peut la déstructurer directement dans les accolades du composant.

Exemple fonctionnel pas à pas

// 1. Création du composant enfant Greeting
function Greeting({ message }) {
  // message est la prop passée depuis le parent
  return <h1>{message}</h1>;
}

// 2. Composant parent App
function App() {
  // On transmet la prop "message" avec une valeur
  return <Greeting message="Bonjour React" />;
}

// 3. Résultat affiché : 
Bonjour React

Explication

1. Greeting reçoit la prop message.
2. La prop est utilisée pour afficher du texte dans le composant.
3. App transmet la valeur "Bonjour React" via l’attribut message. 

<Greeting message="Bonjour React" />

Exercice 2 — Plusieurs props

Passer plusieurs données à un composant.

Théorie

Les props sont des propriétés d’un objet. Elles peuvent être déstructurées pour être utilisées directement.

Exemple fonctionnel pas à pas

// 1. Composant User reçoit deux props : name et age
function User({ name, age }) {
  return <p>Nom: {name}, Age: {age}</p>;
}

// 2. Composant App transmet plusieurs props
function App() {
  return <User name="Lucas" age={25} />;
}

// 3. Résultat affiché : Nom: Lucas, Age: 25

Explication

- name et age sont transmises depuis le parent.
- La déstructuration permet de les utiliser directement sans props.name ni props.age.
- On peut envoyer n’importe quel type de valeur (string, number, boolean, objet…). 

<User name="Lucas" age={25} />

Exercice 3 — Props dynamiques

Utiliser des variables comme props.

Théorie

On peut envoyer des valeurs dynamiques provenant du state ou de variables.

Exemple fonctionnel pas à pas

// 1. Variable dynamique
const username = "Emma";

// 2. Composant User
function User({ name }) {
  return <p>Bonjour {name}</p>;
}

// 3. Composant App
function App() {
  // On transmet la variable username comme prop
  return <User name={username} />;
}

// 4. Résultat affiché : Bonjour Emma

Explication

- La prop name reçoit une valeur dynamique.
- Cela permet d’afficher des informations qui peuvent changer dans l’application. 

<User name={username} />

Exercice 4 — Props children

Insérer du contenu HTML ou des composants dans un composant.

Théorie

La prop spéciale children contient ce qui est placé entre les balises du composant.

Exemple fonctionnel pas à pas

// 1. Composant Card qui reçoit children
function Card({ children }) {
  return <div className="card">
    {children} // Contenu injecté par le parent
  </div>;
}

// 2. Composant App
function App() {
  return (
    <Card>
      <p>Ceci est le contenu de la carte</p>
    </Card>
  );
}

// 3. Résultat affiché : un div contenant le paragraphe

Explication

- children récupère tout ce qui se trouve entre <Card> et </Card>.
- Cela permet d’insérer des textes, images ou autres composants facilement. 

<Card><p>Ceci est le contenu</p></Card>

Exercice 5 — Fonction en prop

Transmettre une fonction pour déclencher une action depuis l’enfant.

Théorie

Les props peuvent contenir des fonctions. L’enfant peut les appeler pour communiquer avec le parent.

Exemple fonctionnel pas à pas

// 1. Composant Button reçoit une fonction handleClick
function Button({ handleClick }) {
  return <button onClick={handleClick}>Cliquez</button>;
}

// 2. Composant parent App définit la fonction
function App() {
  function sayHello() {
    alert("Bonjour !");
  }

  return <Button handleClick={sayHello} />;
}

// 3. Résultat : clic sur le bouton affiche une alerte

Explication

- La prop handleClick contient la fonction du parent.
- Le composant Button déclenche cette fonction au clic.
- Le parent garde le contrôle du comportement. 

<Button handleClick={sayHello} />

Exercice 6 — Liste d’éléments

Générer des composants dynamiquement avec map.

Théorie

map permet de transformer un tableau de données en une liste de composants React.

Exemple fonctionnel pas à pas

// 1. Tableau d'utilisateurs
const users = [
  { id: 1, name: "Lucas", age: 25 },
  { id: 2, name: "Emma", age: 30 }
];

// 2. Composant User
function User({ name, age }) {
  return <p>{name} - {age} ans</p>;
}

// 3. Composant App
function App() {
  return (
    <div>
      {users.map(user => 
        <User key={user.id} name={user.name} age={user.age} />
      )}
    </div>
  );
}

// 4. Résultat : liste de paragraphes pour chaque utilisateur

Explication

- Chaque User reçoit ses props individuelles.
- L’attribut key est nécessaire pour que React identifie les éléments de la liste. 

users.map(user => <User key={user.id} name={user.name} age={user.age} />)
