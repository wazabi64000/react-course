React - useContext (Auth)

← Retour au menu

Le hook useContext permet de partager des données globales dans React. Ici, nous l’utilisons pour gérer un utilisateur authentifié dans toute l’application.

Exercice 1 — Créer un contexte

Initialiser un contexte pour l'utilisateur global.

Théorie

createContext permet de créer un contexte accessible par tous les composants enfants.

Code Exemple

// Création du contexte Auth
import { createContext } from "react";

// AuthContext pourra être partagé
const AuthContext = createContext(null);

import { createContext } from "react"

const AuthContext = createContext(null)

Exercice 2 — Provider

Partager les données utilisateur à tous les composants enfants.

Théorie

Le Provider rend les données accessibles dans l’arborescence React.

Code Exemple

function AuthProvider({ children }) {
  // State pour l'utilisateur
  const [user, setUser] = useState(null);

  return (
    
      {children} {/* rend les enfants avec accès au contexte */}
    
  );
}

function AuthProvider({children}){
 const [user,setUser] = useState(null)

 return (
   <AuthContext.Provider value={{user,setUser}}>
     {children}
   </AuthContext.Provider>
 )
}

Exercice 4 — Logout

Supprimer l’utilisateur du contexte.

Théorie

Code Exemple

Exercice 8 — Token

Stocker un token JWT après login.

Théorie

Le token peut être conservé dans le state ou dans localStorage.

Code Exemple

function login(userData, token) {
  setUser({ ...userData, token });
  localStorage.setItem("token", token); // persistance du token
}

}
